# 🚕 Uber Ride Analytics Dashboard

## Ride Sharing Demand, Revenue & Driver Performance Analytics

---

## 📖 Project Overview

This project analyzes Uber ride booking data to understand customer demand, revenue trends, driver performance, and cancellation behavior.

The dashboard helps identify business insights that can improve driver allocation, reduce cancellations, and enhance customer experience.

---

## ❗ Problem Statement

Ride-sharing companies handle thousands of bookings every day. Managing driver availability, reducing cancellations, and improving customer satisfaction are major challenges.

This project analyzes ride-sharing data to support better business decisions.

---

## 🎯 Objectives

- Identify peak booking hours.
- Analyze revenue by vehicle type.
- Evaluate driver and customer ratings.
- Understand cancellation patterns.
- Analyze payment methods.
- Build an interactive dashboard.

---

## 🛠 Technologies Used

- Python
- Pandas
- Plotly
- Streamlit
- Excel
- SQL

---

## 📊 Dashboard Features

- Total Bookings KPI
- Total Revenue KPI
- Average Ride Distance
- Average Driver Rating
- Peak Booking Hours Analysis
- Revenue by Vehicle Type
- Booking Status Distribution
- Payment Method Analysis
- Driver Rating Analysis
- Customer Rating Analysis
- Customer Cancellation Reasons
- Driver Cancellation Reasons

---

## 📈 Key Insights

- Peak demand occurs during evening hours.
- Premium vehicle types generate higher revenue.
- Digital payment methods are preferred by customers.
- Driver ratings remain above 4.0.
- Driver unavailability contributes to cancellations.
